KGS Calendar Privacy Policy

Last updated: July 7, 2026

Overview

KGS Calendar is a calendar and task app. It has no ads, analytics SDKs, tracking SDKs, or developer-operated backend service.

Data Stored On Your Device

The app stores events, tasks, reminders, settings, source names, server URLs, usernames, and cached calendar data on your device. CalDAV passwords and app passwords are stored with Android encrypted preferences and are excluded from Android app backup.

Android Calendar Access

If you choose to connect Android device calendars and grant calendar permission, the app reads and writes calendar data through the Android Calendar Provider so those calendars can be shown and updated in KGS Calendar. If you do not grant calendar permission or do not add device calendars, this access is not used.

Sync, Accounts, And Subscriptions

If you add a CalDAV or Nextcloud account, the app sends calendar data, task data, server URLs, usernames, and credentials only to the server you configure so syncing can work. If you add a read-only ICS or CalDAV subscription URL, the app requests data from that URL and stores a local cached copy for display. The app does not create a KGS Calendar account.

Location Search And Maps

Location search is optional. When you press Search in the location picker, the search query is sent to OpenStreetMap Nominatim. If you choose nearby results, approximate device location is used to bias that search.

Map previews are loaded from OpenStreetMap after you choose to show a preview. If you enable automatic map previews, the app may load OpenStreetMap data automatically for event and task locations that look like real places.

OpenStreetMap data is © OpenStreetMap contributors.

Backups

Android app backup may include app data according to the device and Android backup settings. The app excludes its encrypted credential preferences from Android cloud backup and device-to-device transfer backup.

Deleting Local Data

You can remove a calendar source in Settings to delete its local credentials and locally stored cached data for that source. Server calendars stay unchanged unless you explicitly use the separate delete-server-calendar action. Uninstalling the app deletes local app data from the device.
