# V.1.3.5 Week View and Widget Event Identity Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an optional week-aligned timeline with configurable whole-week paging, fix independent same-name events being joined in the Month widget, and advance the app to V.1.3.5.

**Architecture:** Keep `CalendarViewMode.ThreeDay` as the persisted timeline destination and derive its presentation from two new DataStore settings. Put week/date arithmetic and pager target selection in pure helpers, then let the existing Compose timeline consume those policies without creating a second pager. Fix the RemoteViews Month widget at its model boundary by joining cells only when occurrence identity and continuation flags both agree.

**Tech Stack:** Kotlin 2.3, Jetpack Compose Foundation pager 1.11, Material 3, DataStore Preferences 1.2.1, Room-backed calendar models, classic RemoteViews widgets, JUnit 4, AndroidX instrumentation tests, Gradle wrapper 9.4.1, JDK 17.

## Global Constraints

- Existing users remain on Multiple days because `weekViewEnabled` defaults to `false`.
- `fullWeekSwipeEnabled` defaults to `true` so newly enabled Week view initially pages by complete weeks.
- Week always renders seven days and honors `firstDayOfWeek` when entered or explicitly navigated.
- Seamless Week entry is week-aligned, but later horizontal gestures may settle one day at a time.
- Keep `CalendarViewMode.ThreeDay` persisted; do not add or migrate to a new enum value.
- Preserve timeline dragging, all-day layout, vertical pinch, current-time rendering, and Day/Multiple days morph behavior.
- Add every user-facing string to both `res/values/strings.xml` and `res/values-de/strings.xml`.
- Month widget rendering remains RemoteViews-based; do not introduce Compose widget code.
- Use `.\gradlew.bat` with JDK 17 and run Gradle commands sequentially.
- Do not read, print, edit, or commit signing secrets, `local.properties`, APKs, AABs, or other build artifacts.
- Set version name to `V.1.3.5` and version code to `15`.

```kotlin
package com.kgs.calendar.domain.model

import java.time.DayOfWeek
import java.time.LocalDate
import java.time.temporal.TemporalAdjusters

const val WEEK_DAY_COUNT = 7

fun LocalDate.startOfWeek(firstDayOfWeek: DayOfWeek): LocalDate =
    with(TemporalAdjusters.previousOrSame(firstDayOfWeek))

fun timelineDayCount(
    viewMode: CalendarViewMode,
    weekViewEnabled: Boolean,
    multiDayCount: Int,
): Int = when (viewMode) {
    CalendarViewMode.Day -> 1
    CalendarViewMode.ThreeDay -> if (weekViewEnabled) WEEK_DAY_COUNT else multiDayCount.coerceMultiDayCount()
    else -> 1
}

fun timelineAnchorDate(
    date: LocalDate,
    viewMode: CalendarViewMode,
    weekViewEnabled: Boolean,
    firstDayOfWeek: DayOfWeek,
): LocalDate = if (viewMode == CalendarViewMode.ThreeDay && weekViewEnabled) {
    date.startOfWeek(firstDayOfWeek)
} else {
    date
}

fun timelineVisibleAnchor(
    date: LocalDate,
    viewMode: CalendarViewMode,
    weekViewEnabled: Boolean,
    fullWeekSwipeEnabled: Boolean,
    firstDayOfWeek: DayOfWeek,
): LocalDate = if (fullWeekSwipeEnabled) {
    timelineAnchorDate(date, viewMode, weekViewEnabled, firstDayOfWeek)
} else {
    date
}

fun moveCalendarPeriod(
    date: LocalDate,
    viewMode: CalendarViewMode,
    delta: Long,
    weekViewEnabled: Boolean,
    multiDayCount: Int,
    firstDayOfWeek: DayOfWeek,
): LocalDate = when (viewMode) {
    CalendarViewMode.Month -> date.plusMonths(delta)
    CalendarViewMode.ThreeDay -> {
        if (weekViewEnabled) {
            date.startOfWeek(firstDayOfWeek).plusWeeks(delta)
        } else {
            date.plusDays(multiDayCount.coerceMultiDayCount().toLong() * delta)
        }
    }
    CalendarViewMode.Day -> date.plusDays(delta)
    CalendarViewMode.Agenda,
    CalendarViewMode.Tasks,
    -> date.plusWeeks(delta)
}
```

Extend `visibleRangeFor` in `CalendarRange.kt` with defaulted settings arguments and use the aligned anchor only for `ThreeDay`:

```kotlin
fun visibleRangeFor(
    date: LocalDate,
    viewMode: CalendarViewMode,
    multiDayCount: Int = DEFAULT_MULTI_DAY_COUNT,
    weekViewEnabled: Boolean = false,
): CalendarRange {
    return when (viewMode) {
        CalendarViewMode.Month -> {
            val monthStart = date.withDayOfMonth(1)
            CalendarRange(monthStart.minusMonths(2), monthStart.plusMonths(3))
        }
        CalendarViewMode.ThreeDay -> CalendarRange(
            date,
            date.plusDays(timelineDayCount(viewMode, weekViewEnabled, multiDayCount).toLong()),
        )
        CalendarViewMode.Day -> CalendarRange(date.minusDays(31), date.plusDays(31))
        CalendarViewMode.Agenda -> CalendarRange(date.minusYears(5), date.plusYears(10))
        CalendarViewMode.Tasks -> CalendarRange(date.minusMonths(1), date.plusMonths(6))
    }
}
```

```kotlin
package com.kgs.calendar.ui.timeline

import com.kgs.calendar.ui.calendar.DayPagerBaseDate
import java.time.DayOfWeek

internal fun weekStartPageOffset(firstDayOfWeek: DayOfWeek): Int =
    Math.floorMod(firstDayOfWeek.value - DayPagerBaseDate.dayOfWeek.value, 7)

internal fun fullWeekTargetPage(
    startPage: Int,
    suggestedTargetPage: Int,
    pageCount: Int,
    weekStartPageOffset: Int,
): Int {
    val alignedStart = startPage - Math.floorMod(startPage - weekStartPageOffset, 7)
    val minimumAlignedPage = weekStartPageOffset.coerceIn(0, pageCount - 1)
    val lastPage = pageCount - 1
    val maximumAlignedPage = lastPage - Math.floorMod(lastPage - weekStartPageOffset, 7)
    val target = when {
        suggestedTargetPage > startPage -> 7
        suggestedTargetPage < startPage -> -7
        else -> 0
    }
    return (alignedStart + target).coerceIn(minimumAlignedPage, maximumAlignedPage)
}
```
